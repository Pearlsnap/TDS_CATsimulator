1. Edit Strands
    StartAbility -- not used. The segment start ability is used instead.
    StartInfo -- also not used.
    MinItems -- the minimum number of operational items required.
    MaxItems -- the maximum number of operational items.
    bpWeight -- the relative weight of this strand to all other strands, content levels and affinity groups in computing the blueprint satisfaction metric. Note that this is not the same as the bpWeight on the segment.
    isStrictMax -- any strand, content level, or affinity group flagged as a strict max will undergo special filtering to attempt to prevent any chance of going over its max items requirement. Over-constrained blueprints however, will find that the algorithm may adjust by violating this constraint if it lacks items to meet other minimums.
    adaptiveCut -- this is generally interpreted as the theta cutpoint between meeting and not meeting proficiency on a strand.
    scalar -- this sets the relative weight of the strand to the other strands in computing the match to ability. Since ability match is only computed on strands and on overall proficiency, this does not apply to any other content levels or affinity groups. A strand can be taken out of the ability computation altogether by making this value zero.
    abilityWeight -- Priority weight associated with a specific reporting category
    precisionTargetNotMetWeight -- Parameter of the function h(.) that controls the information weight when the information target has not yet been hit for reporting category
    precisionTargetMetWeight -- Parameter of the function h(.) that controls the information weight after the information target has been hit for reporting category
    precisionTarget -- Target information for reporting categories
