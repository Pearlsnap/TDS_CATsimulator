    StartAbility -- This is the default ability theta that each student starts at for opportunity 1 unless there is historical data to the contrary. Note that for segmented tests, this value is only applicable to the first segment, unless all preceding segments have only field test items, or operational items that cannot be machine scored. The simulator always assumes that there is historical data and approximates a prior theta score by drawing a random number within +/- 1 of the simulated student's true theta.
    StartInfo -- A seed for the internal information parameter. The default is 0.2 and it is best not to alter this.
    MinItems -- Minimum number of operational items required on the test. (Field test items are specified separately.)
    MaxItems -- Maximum number of operational items allowed on the test.
    bpWeight -- Weight of blueprint satisfaction relative to ability match. Ability match is always 1.0.
    cset1Size -- Out of all item groups in the item pool, this many are passed on to the ability match algorithm. Cset1 consists of the best item groups for satisfying the blueprint at the current point in time (item 'i').
    cset2InitRand -- Cset2 consists of the itemgroups from Cset1 ordered by the linear combination of abilityMatch + bpWeight * blueprintMatch. From this final ordering, one group is randomly chosen. How many of the top itemgroups are in the last cut at the very beginning of the test is determined by cset2InitRand.
    cset2Rand -- Same as cset2InitRand but for every item selection after the first one.
    itemWeight -- If item filtering is done on the itemgroup selected, this weight determines the balance between blueprint and ability metrics. It function the same as bpWeight but only in the context of item filtering.
    abilityOffset -- is added to the ideal item 'b' value in computing ability match. To reduce the b value, make this negative. To increase it, make it positive.
    selectionAlgorithm -- adaptive: run the old adaptive algorithm, adaptive2: run the 2013 version of adaptive algorithm.
    cset1Order -- this is a secondary ordering on the item pool to produce cset1, after blueprint satisfaction. After blueprint satisfaction the algorithm sorts by one of ability match, item distribution, or randomly. Since this is a secondary sort it will only produce an ordering within subsets of itemgroups whose blueprint satisfaction metric is identical. In cases where there are few content levels, there could be very many itemgroups within a single metric. cset1Order of 'ALL' takes all itemgroups greater than or equal to the metric of the group at position cset1Size and passes them all along to the ability match phase.
    ftmin -- minimum number of field test items to administer
    ftmax -- maximum number of field test items to administer
    ftstart -- item position lower bound for field test items
    ftend -- item position upper bound for field test itemsQ
    rcAbilityWeight -- Priority weight associated with reporting category information
    abilityWeight -- Priority weight associated with overall information
    precisionTargetNotMetWeight -- Parameter of the function h(.) that controls the overall information weight when the information target has not yet been hit
    precisionTargetMetWeight -- Parameter of the function h(.) that controls the overall information weight after the information target has been hit
    precisionTarget -- Target information for the overall test
    adaptiveCut -- The overall score cutscore, usually proficiency, used in consideration of TermTooClose
    tooCloseSEs -- The number of standard errors below which the difference is considered too close to the adaptive cut to proceed. In general, this will signal proceeding to a final segment that contains off-grade items
    terminationMinCount -- Flag indicating whether to use minimum test size as a termination condition
    terminationOverallInfo -- Flag indicating whether to use the overall information target as a termination criterion
    terminationRCInfo -- Flag indicating whether to use reporting category information target as a termination criterion
    terminationTooClose -- Flag indicating whether we are not sufficiently distant from the specified adaptiveCut
    terminationFlagsAnd -- Flag indicating whether the termination conditions are to be taken separately or conjunctively
